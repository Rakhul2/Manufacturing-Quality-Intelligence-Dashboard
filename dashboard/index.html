Why drop at a 40% missing threshold instead of, say, 20% or 50%?
The missingness distribution has a natural gap: 32 sensors are missing on 46–91% of units, while every other sensor is missing on 17% or less. Any threshold inside that gap (20%–45%) removes exactly the same 32 sensors, so the choice isn't sensitive to the precise cutoff — it's reading a real structural break in the data, not picking an arbitrary round number.

Why I Built This

A fab line generating 590 sensor readings per unit overwhelms manual root-cause review — engineers need a short, defensible list of sensors to watch, not a spreadsheet of 590 columns. This project builds that list two independent ways: a Random Forest ranks sensors by predictive importance, and a Mann-Whitney U test (with false-discovery-rate correction across all 442 sensors) checks which of those are actually statistically different between pass and fail units. Only sensors that clear both bars are handed to engineering as validated — everything else is reported honestly as suggestive, not proven.

Why not just trust the model's top-15 list?
A Random Forest can assign importance to a sensor because of noise or correlation with a truly-causal sensor, especially with only 104 historical failures. Running an independent statistical test and requiring agreement between the two methods is a cheap, transparent safeguard against reporting a spurious "important" sensor to engineers who'll act on it.

Model Performance

Random Forest, evaluated on a held-out 25% test set never seen during training or feature ranking. Metrics chosen for a ~6.6% failure rate — accuracy would be misleading here.
Why does recall show as 0 at the default threshold, then jump at a "tuned" one?
With balanced class weights but only ~6.6% positive examples, the model never assigns any unit a raw probability above 0.5 — so the default 0.5 cutoff flags nothing. ROC-AUC (0.816) shows the model ranks units by risk well above chance regardless; the fix is picking an operating threshold from the precision-recall curve (0.105 here, chosen to maximize F1), not treating 0.5 as special. In production this threshold would be set by engineering inspection capacity, not by a default.

442 sensors tested — doesn't that risk false positives?
Yes, which is why Benjamini-Hochberg FDR correction is applied across all 442 sensors before checking overlap with the model's top-15, not just to the top-15 in isolation. Testing 442 sensors independently at α=0.05 would produce roughly 22 false positives by chance alone — FDR correction controls that rate directly.
